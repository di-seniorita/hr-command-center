HR-менеджерам каждый день приходится собирать данные из разных мест: таблицы, почта, чаты, рекрутеры. В итоге сложно быстро ответить на простые вопросы: сколько людей на онбординге, кто в зоне риска увольнения, как идёт воронка по кандидатам. HR Command Center — это единый «пульт управления»: один вход, одна система, все ключевые цифры и действия в одном интерфейсе.

Как устроен вход
Сначала HR заходит в систему под своим логином и паролем. После входа открывается личный рабочий стол: дашборд, кандидаты, онбординг, аналитика, отчёты, AI-ассистент, обучение и договоры ГПХ — всё в одном приложении, без переключения между сервисами.

Дашборд
На главном экране сразу видны основные метрики: сколько всего сотрудников, сколько сейчас на онбординге, средняя вовлечённость и процент текучести. Ниже — графики: динамика текучести по месяцам и топ сотрудников с высоким риском ухода. Отдельный блок — алерты: кандидаты, которые давно ждут ответа, сотрудники с высоким риском оттока и просроченные задачи онбординга. Так за пару секунд понятно, куда срочно направить внимание.

Кандидаты и AI-скор
В разделе «Кандидаты» — список по вакансиям с резюме, навыками и статусом (новый, скрининг, интервью, оффер, отказ). У каждого кандидата есть AI-скор — оценка соответствия вакансии по тексту резюме и требованиям (TF-IDF). Можно добавить кандидата вручную или загрузить резюме PDF — система сама извлечёт текст, подставит навыки и пересчитает скор. Статусы можно менять, скор при необходимости пересчитывать по кнопке.

Онбординг и обучение
В «Онбординге» видно сотрудников на адаптации и их задачи: документы, доступы, обучение, знакомство с командой. Задачи можно отмечать выполненными. В «Обучении» — курсы по сотрудникам и их статусы (запланировано, в процессе, завершено). Так видно и процесс входа в компанию, и развитие людей.

Аналитика и риски
В «Аналитике» — сводка по отделам: загрузка, вовлечённость, распределение по департаментам, доля удалённых. Отдельно — риск оттока: модель (логистическая регрессия) учитывает стаж, вовлечённость, удалённость, зарплату и отдел и выдаёт вероятность ухода. Это помогает заранее планировать удержание и разговоры с людьми.

HR-отчёты
Раздел «HR Отчёты» — это готовые визуализации по данным системы: распределение зарплат по отделам, тепловая карта вовлечённости по месяцам, факторы влияния на отток, воронка найма по статусам, сравнение вовлечённости удалённых и офисных. Графики строятся на сервере (matplotlib/seaborn), их можно скачать как PNG и вставить в презентации или отчёты для руководства.

AI-ассистент
В «AI Ассистенте» HR выбирает одного или нескольких кандидатов и задаёт вопрос системе: краткое резюме по кандидату, сравнение по критериям или свой произвольный запрос. Ответ генерирует внешняя языковая модель (LLM): она анализирует резюме и навыки и даёт текстовый вывод. Так можно быстро подготовить выжимку для руководителя или сравнить кандидатов по формулировкам из запроса.

Договоры ГПХ
В разделе «Договоры ГПХ» — список подрядчиков с часами, ставками и статусами (черновик, согласован, подписан). Можно создавать договоры и генерировать акты выполненных работ по шаблону — всё в одном месте с остальными HR-процессами.

HR Command Center объединяет найм, онбординг, аналитику, отчётность и работу с AI в одном приложении. Цель — не заменить живую работу HR, а дать им быстрый доступ к цифрам и подсказкам: кто в риске, кого срочно обработать, как выглядят ключевые метрики и отчёты. Система показывает, как современный HR-инструмент может выглядеть: с авторизацией, дашбордом, AI-скором резюме, прогнозом оттока и AI-ассистентом для работы с кандидатами — всё в одном интерфейсе, без лишних переключений.
Так же к финалу могу сделать модуль сравнения резюме с помощью ИИ с визуальным отображением самих резюме.# HR Command Center



Полноценное веб-приложение для HR-менеджеров с аналитикой и AI-модулями:
- AI-скоринг кандидатов (TF-IDF + cosine similarity)
- Прогноз риска оттока сотрудников (Logistic Regression)
- Дашборды, онбординг, обучение, договоры ГПХ

## Стек

### Backend
- Python 3.11+
- FastAPI
- SQLAlchemy
- SQLite
- scikit-learn

### Frontend
- React 18 + Vite
- Tailwind CSS
- Recharts

## Структура проекта

```text
/backend   -> FastAPI API, модели БД, сиды, AI-модули
/frontend  -> React приложение (дашборд и страницы HR)
```

## 1) Запуск backend

Откройте терминал в корне проекта и выполните:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r backend/requirements.txt
uvicorn backend.main:app --reload --host 0.0.0.0 --port 8000
```

Backend будет доступен по адресу:
- API: http://localhost:8000
- Swagger: http://localhost:8000/docs

> При первом запуске автоматически создаются таблицы SQLite и заполняются реалистичными тестовыми данными на русском языке.

## 2) Запуск frontend

В новом терминале:

```powershell
cd frontend
npm install
npm run dev
```

Frontend будет доступен по адресу:
- http://localhost:5173

## AI-модули

### 1. Resume Scoring (`backend/ai_scoring.py`)
- Строит TF-IDF по двум документам:
  - текст кандидата (`resume_text + skills`)
  - текст вакансии (`required_skills_text + description`)
- Возвращает score от 0 до 100
- Автоматически вызывается при создании кандидата
- Можно пересчитать через `GET /api/candidates/{id}/rescore`

### 2. Churn Prediction (`backend/ai_churn.py`)
- Модель: `LogisticRegression`
- Признаки:
  - `tenure_months`
  - `avg_engagement`
  - `is_remote`
  - `salary`
  - `department` (через `LabelEncoder`)
- Обучение выполняется при старте backend на seed-данных
- Прогноз доступен через `GET /api/analytics/churn-risk`

## Примечания по безопасности и качеству

- Включен CORS только для `http://localhost:5173`
- Выполнена серверная валидация статусов и категорий
- На backend используются типизированные Pydantic-модели
- Все вычисления сумм по договорам выполняются на сервере
